Module Appointments

module Appointments: sig .. end

Representation of an appointment book

This module represents the data stored in a digital appointment book, including the data for each individual appointment.

type app

app is the type that represents a singular appointment, including all of the data that is associated with the appointment.

val empty_appo : unit -> app list Stdlib.ref

empty_appo () creates an empty appointment book. Upon creation, there is no appointments or data whatsoever in the appointment book.

val access_app : ?appo:app list Stdlib.ref -> unit -> app list

access_app () unlocks and opens the appointment book so all appointments and data associated with them are visible.

val delete_app : ?appo:app list Stdlib.ref -> string -> unit

delete_app t finds the appointment with the name t and deletes that appointment from the appointment book.

val add_app : ?appo:app list Stdlib.ref -> string -> string -> string -> unit

add_app t d i adds an appointment to the digital appointment book. For an appointment to be added, a name t, date d, and time i for the appointment is required.

val complete_app : ?appo:app list Stdlib.ref -> string -> unit

complete_app t completes an appointment with name t by deleting the appointment from the appointment book.

val add_app_info : ?appo:app list Stdlib.ref -> string -> string -> unit

add_app_info t i adds information i to the data of the appointment with name t.

val add_location : ?appo:app list Stdlib.ref -> string -> string -> unit

add_location t l adds the location l of the appointment with name t to the data of the appointment with name t.

val to_list_app : ?appo:app list Stdlib.ref -> string list -> string list

to_list_app a creates a list representation with each appointment name and all the data associated with each appointment.

val empty_finder : unit -> app list Stdlib.ref

empty_finder () creates a pointer to assist in finding the specific appointment a user wants.

val find_app_user : ?one:app list Stdlib.ref ->
       ?appo:app list Stdlib.ref -> string -> unit

find_app_user t finds the appointment with name t and its associated data.

val to_list_find : ?one:app list Stdlib.ref -> string list -> string list
